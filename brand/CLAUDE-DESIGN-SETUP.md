## 2. 실행 절차 — 이 방법을 권장

### ✅ 방법 A · Claude Code에서 `/design-sync` (권장)

1. **이 세션(design)에 그냥 말한다**: `design-sync 해줘`
2. Claude가 이렇게 진행한다:
   - `list_projects` — 기존 디자인 시스템 프로젝트 확인 *(2026-07-17 기준: **없음**. 새로 만들어야 함)*
   - `create_project` — `사르르목장` 이름으로 디자인 시스템 프로젝트 생성 → **승인 프롬프트 뜸**
   - `finalize_plan` — 올릴 파일 목록을 확정해서 보여줌 → **승인 프롬프트 뜸**
     - 이때 **어떤 파일이 어디로 올라가는지 화면에 그대로 표시**된다. 확인하고 승인.
   - `write_files` — 업로드
3. **claude.ai/design 에서 확인** — 새 프로젝트가 보이고, Design System 패널에 카드가 뜬다.

> **왜 이게 나은가**: 붙여넣기는 사람이 실수하고, 나중에 토큰이 바뀌면 또 붙여넣어야 한다.
> design-sync는 **증분 동기**라 바뀐 것만 올라간다.

### 방법 B · GitHub 레포로 가져오기

Claude Design이 GitHub 레포에서 디자인 시스템을 가져올 수 있다.
우리 브랜드 패키지는 이미 공개 레포에 있다:

```
https://github.com/jaehwan-lee-benja/saruru-design-dashboard
→ brand/ 폴더
```

claude.ai/design 에서 디자인 시스템 추가 시 이 레포를 지정하고 `brand/` 경로를 가리키면 된다.
**(⚠ 화면에서 확인 필요 — 경로 지정 UI가 있는지, 레포 전체만 되는지 문서에 없음)**

### 방법 C · 수동 업로드 (최후)

`brand/tokens.css` · `brand/tokens.json` · `brand/components.html` 3개 파일을 업로드.
**단 `assets/mascot.png`·`logo.png`의 상대경로가 깨지므로 이미지도 같이 올려야 한다.**

## 4. ⚠ 밀어넣기 전에 결정할 것 — 딱 하나

### 네이비 CMYK 2안 중 택1

리플렛 `.ai`에서 **네이비 CMYK가 2개** 나왔고, 어느 쪽이 로고의 것인지 **아직 모른다.**

| | CMYK | 사용 빈도 | 렌더 |
|---|---|---|---|
| **A안** | `C93 M74 Y19 K5` | 10회 (주) | `#304d8f` |
| **B안** | `C87 M69 Y20 K4` | 6회 (부) | `#3a5492` |

- 화면 RGB `#38528a`는 **확정**이다(로고 PNG 실측). 이건 안 건드려도 된다.
- **인쇄용 CMYK만 미정**이라, 지금 밀어넣어도 화면 작업엔 지장 없다.
- **다만 대형 사인·간판 발주 전에는 반드시 정해야 한다.** 여기서 틀리면 간판 색이 어긋난다.

**푸는 법**: Illustrator에서 `사르르목장_메인로고_인쇄용.ai`를 열고 → 로고 패스 선택 →
색상 패널에서 CMYK 값 확인. 그 값이 정답이다. **1분이면 된다.**

## 5. ★한글 서체 — 여기가 제일 까다롭다

**브랜드 한글 = `Sandoll 오동통` (산돌 · 유료 · 산돌구름 구독)**

### 규칙
- ❌ **폰트 파일(.otf)을 업로드하지 말 것.** 라이선스가 있어도 **재배포는 별개**다.
- ✅ **`font-family`로 이름만 참조**한다 → 라이선스 보유 기기(작업용 맥)에서만 정품 렌더.
  `tokens.css`가 이미 그렇게 돼 있다:
  ```css
  --saruru-font-ko: 'Sandoll 오동통', 'SDOdongtongBasic', 'Cafe24Ssurround', sans-serif;
  ```

### Claude Design에서 예상되는 현상
- **작업용 맥의 브라우저에서 열면** → 정품 오동통으로 보인다. ✅
- **Claude가 서버에서 렌더/스크린샷하면** → 오동통이 없으니 **카페24 써라운드로 폴백**될 것이다.
  → 글자 모양이 실제와 다르게 보일 수 있다. **레이아웃만 믿고 글자 모양은 내 화면에서 확인할 것.**
- **(화면에서 확인 필요)** Claude Design이 폰트를 어떻게 처리하는지는 문서에 없다.

### 라이선스 구조 (알아두면 좋은 것)
- 영문 **Fredoka = SIL OFL(영구 무료)** ↔ 한글 **오동통 = 유료 구독**
  → **서체 시스템이 이원 구조**다. 구독이 끊기면 **한글만** 막힌다.
- **★단, `(글씨깸)` 처리한 발주본은 구독과 무관하다** — 글자가 패스라 폰트가 아니기 때문.
  → **아카이브는 안전하고, 위험한 건 "신규 조판"이다.** 지금 하고 있는 글씨깸 관행이 사실상 보험이다.

## 6. 브랜드 규칙 요약 (Claude가 지켜야 할 것)

토큰만으로는 전달 안 되는, **말로 해야 하는 규칙**이다. 프로젝트 지침에 붙여넣기 좋게 정리했다.

```
사르르목장 브랜드 규칙

[정본] 물리·인쇄물 = 2021 STUDIO FLAT FLAG 라인아트(정본 A).
       픽셀 BI는 게임·대시보드 등 디지털 전용. 둘을 섞지 않는다.

[컬러] 네이비 #38528a 가 기본.
       그린 #45bc51 은 "이것이 핵심"일 때만 — 한 화면에 한 곳. 남발 금지.
       빨강은 브랜드 색이 아니다. 법정 금지 픽토그램(ISO 7010)에만 예외 허용,
       타이포에는 절대 금지.
       기본 지면은 화이트. 크림 #f5f3ef 은 패널 구획용이지 지면색이 아니다.

[선]   균일 굵기 라인아트, 둥근 캡. 굵기는 글자 획에 시각적으로 맞춘다.
       사실적 명암·그라데이션·복잡한 디테일 금지.

[화법] 명령하지 말고 이유를 준다. 질문형으로 던지고 답한다.
       3키워드: 고소 · 신선 · 담백. 슬로건: 행복이 사르르.
       예) "왜 이렇게 신선한가요? → 당일 착유하기 때문입니다"
       단 법정 표지는 예외 — 명확히 고지한다.

[마스코트] 젖소 3마리(소프트콘·우유팩·스쿱). 메인로고에 내장돼 분리 불가.
       절제해서 쓴다 — 한 곳에 집중. 법정 표지에는 쓰지 않는다.

[톤]   레이아웃·타이포·여백은 "세련된 문화센터" — 정렬 엄격, 여백 넉넉, 장식 절제.
       귀여움은 마스코트·서체가 담당. 절제된 판 위의 귀여움이 더 귀엽다.
```

## 7. 안 되는 것 / 한계

- **Figma 연동**: 문서에 언급 없음. 우리는 Figma를 안 쓰므로 무관.
- **`.ai` 원본**: Claude Design이 Illustrator 파일을 읽는다는 근거 없음.
  실제 발주본은 **Illustrator에서 사람이** 만든다. Claude Design은 **시안·화면 작업용**이다.
- **인쇄 규격**: 토큰에 담기지 않는다. 칼선 규격 12종은 `../topics.js` C03 참조.
- **자동 상속의 범위**: "프로젝트 생성 시 조직 디자인 시스템 자동 상속"이라고만 돼 있음.
  1인 계정에서 "조직"이 어떻게 동작하는지는 **화면에서 확인 필요**.
