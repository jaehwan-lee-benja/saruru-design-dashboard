# saruru-design-dashboard

사르르목장 **디자인 아카이브** 대시보드 (정적 · 로그인 전용).

- 단일 `index.html`. 빌드 없음. GitHub Pages(main 루트) 배포 — push = 배포.
- **로그인 필수** — Google OAuth(Supabase multi-store `rstazttwlghsorpzsugy`).
  등록 관리자 계정(`designerbenja` · `kbl0226`)만 열람. 현재는 클라이언트 이메일 게이트
  (crm/game-dashboard 선례와 동일. 실데이터 연동 시 RLS로 승격).
- 이 저장소는 **표시(프론트)만** 담는다. 원본 작업물은 Google Drive에 있고, 대시보드는 링크만 건다.
- 통일 규약: 도메인별 대시보드 = `github.io/<domain>-dashboard/` + Supabase 로그인.

라이브: https://jaehwan-lee-benja.github.io/saruru-design-dashboard/

## 무엇을 하는가

2022년부터 4년간 **그때그때 만들어져 통일되지 않은** 디자인 작업물(~150개 `.ai`)을
**기능 기준으로 분류**하고, 각 항목의 **비포(현재 원본 Drive 링크)** ↔ **애프터(개선본)** 를 추적한다.

## 분류 체계

**기능 1차 + 형식 2차(태그).** 근거·상세는 정본 문서 `saruru-design/ARCHIVE-TAXONOMY.md`.

| No | 카테고리 | 우선순위 |
|---|---|---|
| 01 | BI · 브랜드 코어 | 정본(개선 대상 아님) |
| 02 | 사이니지 · 매장 안내 체계 | ★1순위 |
| 03 | 메뉴 · 가격 | 2순위 |
| 04 | 패키지 · 라벨 | 3순위 |
| 05 | 홍보 · 캠페인 | 4순위 |
| 06 | 공간 조형 · 시즌 | 5순위 |

02는 하위 기능(방향 / 운영 / 주의·금지 / 시설 / 시즌·농사)으로 한 번 더 나뉜다.

## 브랜드 준수

- **정본 A = 2021 STUDIO FLAT FLAG 라인아트** (유저 확정 2026-07-17). 픽셀 BI는 디지털 전용.
- 네이비 **`#38528a`** — 원본 메인로고 픽셀 실측값.
- 보조 그린 **`#45bc51`** = **강조 키워드 전용**(남발 금지). 리플렛 300dpi 실측.
- 인쇄: **별색 없음 · 4도 프로세스** 확정. ⚠**네이비 CMYK는 미확정** — 리플렛에 네이비 CMYK가 2개(`C93 M74 Y19 K5` / `C87 M69 Y20 K4`)이고 어느 쪽이 로고의 것인지 아직 모름. 대형 사인 발주 전 반드시 정리.
- 상세: `saruru-design/DESIGN-SYSTEM.md`

## 애프터 작업 순서

유저 지시: **대시보드 + 목차 + 비포 링크 먼저 → 애프터는 순차.**
애프터 완료 시 `index.html`의 해당 item에 `a:'<링크>'` 추가 + `st:'애프터'`로 변경하면 자동 반영.
